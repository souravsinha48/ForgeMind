## 1. Project Overview

This capstone builds a live terminal-based factory simulation where an AI-powered agent pipeline predicts machine failures before they happen and dispatches corrective actions in real time. A CNN-LSTM deep learning model, trained on NASA's N-CMAPSS turbofan engine degradation dataset, serves as the predictive oracle. Three cooperative AI agents — a diagnostic translator, a deterministic capacity calculator, and an LLM-powered floor manager — chain together to convert a professor's plain-English chaos input ("bearing temperature spike on Machine 4") into a sensor-level fault injection, a Remaining Useful Life (RUL) prediction, a factory-wide capacity recalculation, and a strategic dispatch order, all rendered in a four-pane Textual terminal dashboard that updates live.

### Academic Context

The project spans three MIOM disciplines: deep learning (CNN-LSTM time-series regression), operations management (capacity planning with ΣPD/T break-even analysis), and agentic AI (multi-agent orchestration with structured tool use). The terminal "game" format allows a live professor-driven demo where fault scenarios are injected on the fly.

### The NASA-to-Factory Conceptual Bridge

N-CMAPSS simulates commercial turbofan jet engines, not factories. The translation is an explicit, defensible metaphor:

| N-CMAPSS Concept | Factory Fiction | Why It Works |
|---|---|---|
| One engine unit (e.g., Unit 2) | One CNC machine / compressor | Both are rotating machinery with wear-based degradation |
| Flight cycle (takeoff → cruise → land) | Production shift cycle (ramp-up → full load → cooldown) | Same stress-and-recover degradation pattern |
| Flight phases (Mach, altitude, TRA) | Operating conditions (load setting, ambient temp, speed) | Both define the stress regime |
| Physical sensors (Ps30, T50, Nf, …) | Factory sensors (bearing temp, vibration, RPM, pressure) | Functionally identical measurements |
| RUL in flight cycles | RUL in production shift-cycles | Same integer output, different label |
| Fault modes (DS01–DS08) | Fault scenarios the professor "injects" | Multiple fault types = multiple chaos scenarios |
| Engine-to-engine variation | Machine-to-machine variance | Justifies why Machine 3 degrades faster than Machine 1 |

The dataset itself is already simulated (generated by NASA's CMAPSS tool running real flight profiles). No synthetic data generation is needed on top of it. The 9 engine units in DS02 become 9 factory machines; we select 4–5 for the active factory floor.

## 5. Deep Learning Specification

### 5.1 Dataset: N-CMAPSS DS02

**File:** `N-CMAPSS_DS02-006.h5` from NASA's prognostic data repository.

Use DS02 (data-driven prognostics), not DS01 (model-based diagnostics).

#### HDF5 Schema

| HDF5 Key | Contents | Shape |
|---|---|---|
| `W` | Operating conditions (Mach, altitude, TRA, T2) | `(N, 4)` |
| `X_s` | Physical sensor readings (14 sensors) | `(N, 14)` |
| `X_v` | Virtual sensor readings (14 sensors) — **not used** | `(N, 14)` |
| `T` | Health state auxiliary variables | `(N, 4)` |
| `Y` | RUL ground truth labels | `(N, 1)` |
| `A` | Unit/cycle metadata | `(N, 5)` |

**Model input:** `X_s` (14 physical sensors) + `W` (4 operating conditions) = **18 features**.

Virtual sensors (`X_v`) are excluded — they are model-derived and add complexity without added DL challenge.

#### Train / Test Split

| Set | Engine Units |
|---|---|
| Training | 2, 5, 10, 16, 18, 20 (6 units) |
| Test | 11, 14, 15 (3 units) |

class NCMAPSSDataset(Dataset):
    def __init__(self, X, Y, window=50, stride=1):
        self.samples, self.labels = [], []
        for i in range(0, len(X) - window, stride):
            self.samples.append(X[i:i+window])
            self.labels.append(Y[i+window-1])

        self.labels  = np.array(self.labels,  dtype=np.float32)

**Memory tip:** Use `--sampling 10` (1 sample every 10 seconds = 0.1 Hz) to cut memory by 10× with minimal accuracy loss. Validate pipeline at `--sampling 10` first, then run full 1 Hz on H100 if time permits.

### 5.3 Model Architecture: CNN-LSTM

```python
class CNNLSTM_RUL(nn.Module):
    def __init__(self, n_features=18, window=50, cnn_filters=64,
                 lstm_hidden=128, lstm_layers=2, dropout=0.3):
        super().__init__()
        self.cnn = nn.Sequential(
            nn.Conv1d(n_features, cnn_filters, kernel_size=3, padding=1),
            nn.BatchNorm1d(cnn_filters),
            nn.ReLU(),
            nn.Conv1d(cnn_filters, cnn_filters * 2, kernel_size=3, padding=1),
            nn.BatchNorm1d(cnn_filters * 2),
            nn.ReLU(),
            nn.Dropout(dropout)
        )
        self.lstm = nn.LSTM(
            input_size=cnn_filters * 2, hidden_size=lstm_hidden,
            num_layers=lstm_layers, batch_first=True, dropout=dropout
        )
        self.regressor = nn.Sequential(
            nn.Linear(lstm_hidden, 64), nn.ReLU(),
            nn.Dropout(dropout), nn.Linear(64, 1)
        )

    def forward(self, x):
        x = x.permute(0, 2, 1)          # (B, 50, 18) → (B, 18, 50)
        x = self.cnn(x)                  # → (B, 128, 50)
        x = x.permute(0, 2, 1)          # → (B, 50, 128)
        _, (h_n, _) = self.lstm(x)      # h_n: (2, B, 128)
        x = h_n[-1]                      # → (B, 128)
        return self.regressor(x).squeeze(-1)  # → (B,)
```

#### Tensor Shape Trace

| Stage | Shape | Notes |
|---|---|---|
| Input | `(B, 50, 18)` | Batch × window × features |
| After permute | `(B, 18, 50)` | Conv1d expects (B, C, L) |
| After CNN block | `(B, 128, 50)` | 128 = `cnn_filters * 2` |
| After permute back | `(B, 50, 128)` | LSTM expects (B, L, C) |
| LSTM hidden (h_n) | `(2, B, 128)` | 2 = `lstm_layers` |
| Last layer hidden | `(B, 128)` | Take `h_n[-1]` |
| After regressor | `(B,)` | Single RUL float per sample |

**Why BatchNorm1d?** N-CMAPSS has significant intra-unit variance across flight phases. BatchNorm stabilizes training and accelerates convergence on this dataset.

### 5.4 Training Configuration

| Parameter | Value | Notes |
|---|---|---|
| Window | 50 | 50-timestep sliding window |
| Batch size | 512 | H100 can handle 1024+; 512 is stable |
| Learning rate | 1e-3 | Adam optimizer |
| Epochs | 50 (max) | With early stopping |
| Loss function | MSELoss | Standard for RUL regression |
| Scheduler | ReduceLROnPlateau | patience=5, factor=0.5 |
| Early stopping | 10 epochs | Stop if val_loss stalls for 10 epochs |
| Dropout | 0.3 | Applied in CNN and regressor |
| LSTM hidden | 128 | 2-layer LSTM |
| CNN filters | 64 | Doubles to 128 in second conv layer |

**Evaluation metrics:**
- **Primary:** RMSE = √(MSE) — penalizes large errors
- **Secondary:** Asymmetric scoring function — late predictions penalized more heavily than early

**Expected training time on H100:** ~3–5 min/epoch → 50 epochs ≈ 3–4 hours total, within the 12-hour Kaggle session cap.

### 5.7 Inference API — The Contract File

```python
# dl_engine/inference.py — this is ALL other teams ever import

import torch, joblib, numpy as np
from .model import CNNLSTM_RUL

_model  = None
_scaler = None

def load_model(weights_path='dl_engine/weights/best_model.pt',
               scaler_path='dl_engine/weights/scaler.pkl'):
    global _model, _scaler
    checkpoint = torch.load(weights_path, map_location='cpu')
    _model = CNNLSTM_RUL()
    _model.load_state_dict(checkpoint['model_state_dict'])
    _model.eval()
    _scaler = joblib.load(scaler_path)

def predict_rul(sensor_tensor: np.ndarray) -> float:
    """
    Input:  numpy array of shape (50, 18) — one window of scaled sensor data
    Output: float RUL prediction (in production shift-cycles)
    """
    if _model is None:
        load_model()
    scaled = _scaler.transform(sensor_tensor)
    t = torch.tensor(scaled, dtype=torch.float32).unsqueeze(0)  # (1, 50, 18)
    with torch.no_grad():
        rul = _model(t).item()
    return max(0.0, rul)  # RUL cannot be negative
```

## 6. Agent Specification

### 6.0 Input Guard — Nonsensical Input Rejection

**Problem:** The professor (or anyone demoing) might type something completely off-topic — "the coffee machine is broken", "what if there's an earthquake", or just mash the keyboard. Passing garbage to Agent 1's Gemini call wastes time and produces a meaningless sensor spike that corrupts the simulation.

**Solution:** A lightweight pre-LLM filter that runs before Agent 1. This is **not** an LLM call — it's a fast keyword/heuristic check.

```python
# agents/input_guard.py

# Keywords that indicate a valid factory fault input
VALID_KEYWORDS = {
    "temperature", "temp", "pressure", "vibration", "bearing", "rpm", "speed",
    "surge", "spike", "leak", "overheat", "overload", "failure", "fault",
    "sensor", "coolant", "lubrication", "friction", "wear", "crack",
    "misalignment", "imbalance", "corrosion", "degradation", "noise",
    "machine", "cnc", "press", "lathe", "mill", "compressor", "motor",
    "pump", "valve", "filter", "shaft", "rotor", "stator", "gearbox",
}

def is_valid_fault_input(user_text: str) -> tuple[bool, str]:
    """
    Returns (is_valid, reason).
    If invalid, reason explains why — this gets shown in the comms log.
    """
    text_lower = user_text.strip().lower()

    if len(text_lower) < 5:
        return False, "Input too short. Describe a machine fault (e.g., 'bearing temperature spike on Machine 4')."

    if len(text_lower) > 500:
        return False, "Input too long. Keep fault descriptions under 500 characters."

    # Check for at least one factory/fault-related keyword
    words = set(text_lower.split())
    if not words.intersection(VALID_KEYWORDS):
        return False, (
            f"Unrecognized fault type. Try something like: "
            f"'high pressure in compressor', 'bearing overheat on Machine 3', "
            f"or 'vibration spike on CNC-Alpha'."
        )

    return True, ""
```

**Usage in agent_loop.py:**

```python
from agents.input_guard import is_valid_fault_input

valid, reason = is_valid_fault_input(user_text)
if not valid:
    log_to_comms(f"[Chaos Engine] REJECTED: {reason}")
    return  # Do not call Agent 1
```

The keyword set is intentionally broad — it catches obvious nonsense while allowing creative fault descriptions. If the professor consistently hits the guard with valid-sounding inputs, expand the keyword set.


### 6.1 Agent 1 — Diagnostic Agent

**Purpose:** Translates a professor's natural-language fault description into a concrete sensor-level spike injected into a base sensor window.

**LLM:** Gemini 2.5 Flash

**Input:** `base_window: np.ndarray` (shape `(50, 18)`) + `user_text: str`

**Output:** Modified `np.ndarray` (shape `(50, 18)`) with fault injected, plus the parsed spike metadata.

#### Required JSON Response Schema

The Gemini call must return **only** a valid JSON object with this structure:

```json
{
  "sensor_id": "Xs4",
  "spike_value": 0.97,
  "affected_window_positions": [45, 46, 47, 48, 49],
  "fault_severity": "HIGH",
  "plain_english_summary": "Bearing temperature sensor spiked to near-maximum in final 5 timesteps."
}
```

| Field | Type | Constraints | Description |
|---|---|---|---|
| `sensor_id` | string | One of `W0`–`W3` or `Xs0`–`Xs13` (18 valid values) | Sensor name |
| `spike_value` | float | `0.0 ≤ v ≤ 1.0` | Injected value in normalized range |
| `affected_window_positions` | int[] | Each element `0 ≤ p ≤ 49`, max 10 positions | Timestep positions to modify |
| `fault_severity` | string | One of `"LOW"`, `"MEDIUM"`, `"HIGH"` | Severity tag |
| `plain_english_summary` | string | Non-empty | One-sentence description for comms log |

#### Validation, Retry, and Deterministic Fallback

LLMs are stochastic. Gemini will occasionally wrap JSON in backticks, hallucinate sensor IDs, or return positions out of range. **Never trust raw LLM output in a live demo.**

def _strip_markdown_fences(text: str) -> str:
    """Remove ```json ... ``` wrapping that Gemini sometimes adds."""
    text = text.strip()
    text = re.sub(r'^```(?:json)?\s*', '', text)
    text = re.sub(r'\s*```$', '', text)
    return text.strip()

def _validate_spike(spike: dict) -> bool:
    """Validate all fields are within legal bounds."""
    if spike.get('sensor_id') not in VALID_SENSORS:
        return False
    if not (0.0 <= spike.get('spike_value', -1) <= 1.0):
        return False
    positions = spike.get('affected_window_positions', [])
    if not positions or not all(0 <= p <= 49 for p in positions):
        return False
    if len(positions) > 10:
        return False
    if spike.get('fault_severity') not in VALID_SEVERITIES:
        return False
    if not spike.get('plain_english_summary'):
        return False
    return True

def _get_fallback(user_text: str) -> dict:
    """Keyword match to pick the best deterministic fallback."""
    text_lower = user_text.lower()
    for keyword, spike in FALLBACK_SPIKES.items():
        if keyword in text_lower:
            return spike.copy()
    return FALLBACK_SPIKES["default"].copy()

def translate_fault_to_tensor(base_window: np.ndarray, user_text: str) -> tuple[np.ndarray, dict]:
    spike = None

    for attempt in range(MAX_RETRIES + 1):
        try:
            response = client.models.generate_content(
                model="gemini-2.5-flash",
                contents=f"{SYSTEM_PROMPT}\n\nFault description: {user_text}"
            )
            raw = _strip_markdown_fences(response.text)
            candidate = json.loads(raw)
            if _validate_spike(candidate):
                spike = candidate
                break
        except (json.JSONDecodeError, KeyError, TypeError, Exception):
            continue  # Retry

    # If all retries failed, use deterministic fallback
    if spike is None:
        spike = _get_fallback(user_text)
        spike['plain_english_summary'] += " [FALLBACK — LLM unavailable]"

    # Inject the spike into the sensor window
    injected = base_window.copy()
    sid = spike['sensor_id']
    if sid.startswith('Xs'):
        sensor_idx = int(sid.replace('Xs', '')) + 4
    else:
        sensor_idx = int(sid.replace('W', ''))
    for pos in spike['affected_window_positions']:
        injected[pos][sensor_idx] = spike['spike_value']

    return injected, spike
```

### 6.2 Agent 2 — Capacity Agent

**Purpose:** Deterministic MIOM capacity recalculation. **Zero LLM calls — pure Python math.**

#### Machine Parameters

| ID | Name | Base Time (hrs) | Product Demand (units) |
|---|---|---|---|
| 1 | CNC-Alpha | 8.0 | 120 |
| 2 | CNC-Beta | 8.0 | 95 |
| 3 | Press-Gamma | 8.0 | 140 |
| 4 | Lathe-Delta | 8.0 | 110 |
| 5 | Mill-Epsilon | 8.0 | 130 |

#### Three-State Machine Status (Replaces Binary ON/OFF)

The original design used a hard cliff: RUL ≤ 15 → instant OFFLINE. This creates an unrealistic discontinuity where a machine at RUL=16 contributes full capacity and at RUL=15 contributes zero. A three-state model produces more realistic and visually interesting degradation during the demo.

| RUL Range | Status | Available Time | Rationale |
|---|---|---|---|
| RUL > 30 | **ONLINE** | 100% of base time (8.0 hrs) | Healthy — full production |
| 15 < RUL ≤ 30 | **DEGRADED** | 50% of base time (4.0 hrs) | Reduced speed for safety; maintenance window opening |
| RUL ≤ 15 | **OFFLINE** | 0% (0.0 hrs) | Mandatory shutdown for preventive maintenance |

```python
# agents/capacity_agent.py — NO LLM CALLS IN THIS FILE, EVER

MACHINES = {
    1: {"name": "CNC-Alpha",   "base_time": 8.0, "product_demand": 120, "available_time": 8.0},
    2: {"name": "CNC-Beta",    "base_time": 8.0, "product_demand": 95,  "available_time": 8.0},
    3: {"name": "Press-Gamma", "base_time": 8.0, "product_demand": 140, "available_time": 8.0},
    4: {"name": "Lathe-Delta", "base_time": 8.0, "product_demand": 110, "available_time": 8.0},
    5: {"name": "Mill-Epsilon","base_time": 8.0, "product_demand": 130, "available_time": 8.0},
}

RUL_OFFLINE_THRESHOLD  = 15   # cycles — hard shutdown
RUL_DEGRADED_THRESHOLD = 30   # cycles — reduced capacity

def update_capacity(machine_id: int, new_rul: float) -> dict:
    machine = MACHINES[machine_id]

    if new_rul <= RUL_OFFLINE_THRESHOLD:
        machine['available_time'] = 0.0
        status = "OFFLINE"
    elif new_rul <= RUL_DEGRADED_THRESHOLD:
        machine['available_time'] = machine['base_time'] * 0.5
        status = "DEGRADED"
    else:
        machine['available_time'] = machine['base_time']
        status = "ONLINE"

    total_T    = sum(m['available_time'] for m in MACHINES.values())
    total_PD   = sum(m['product_demand'] for m in MACHINES.values())
    machine_req = total_PD / total_T if total_T > 0 else float('inf')
    capacity_pct = (total_T / (len(MACHINES) * 8.0)) * 100

    return {
        "machine_id":      machine_id,
        "status":          status,
        "rul":             new_rul,
        "total_T":         round(total_T, 2),
        "total_PD":        total_PD,
        "machine_req":     round(machine_req, 3),
        "capacity_pct":    round(capacity_pct, 1),
        "breakeven_risk":  machine_req > 1.0
    }
```

#### MIOM Formulas

```
Total Available Time (T)    = Σ machine.available_time   (hours)
Total Product Demand (ΣPD)  = Σ machine.product_demand   (units)
Machine Requirement Ratio   = ΣPD / T                    (units/hour)
Capacity Percentage         = (T / (num_machines × 8.0)) × 100
Break-even Risk             = TRUE if (ΣPD / T) > 1.0
```

#### Output Schema

```python
{
    "machine_id":      int,          # Which machine was updated
    "status":          str,          # "ONLINE", "DEGRADED", or "OFFLINE"
    "rul":             float,        # Current RUL prediction
    "total_T":         float,        # Total available hours
    "total_PD":        int,          # Sum of all product demands
    "machine_req":     float,        # ΣPD / T ratio
    "capacity_pct":    float,        # Factory capacity percentage
    "breakeven_risk":  bool          # True if machine_req > 1.0
}
```

### 6.3 Agent 3 — Floor Manager Agent

**Purpose:** Receives the deterministic capacity report and issues human-readable dispatch orders.

**LLM:** Gemini 2.5 Flash

**Input:** The capacity report dict from Agent 2.

**Output:** A short terminal-style dispatch string (max 4 sentences).

#### System Prompt Rules

1. **NEVER** invent or modify any number — use only figures given in the report.
2. Issue **specific, actionable** orders (reroute production, authorize overtime, schedule maintenance).
3. Maximum **4 sentences**. Terminal-style. No bullet points.
4. Begin every response with `[Floor Manager]`.
5. For **DEGRADED** machines: recommend load reduction and schedule a maintenance window.
6. For **OFFLINE** machines: reroute production to remaining machines and escalate.

#### Context Template Fed to Gemini

```
Machine {machine_id} ({machine_name}) is now {status}.
RUL = {rul:.1f} cycles.
Factory capacity: {capacity_pct}%.
Total available time (T): {total_T} hours.
Sum of Product Demand (ΣPD): {total_PD} units.
Machine Requirement ratio (ΣPD/T): {machine_req}.
Break-even risk: {'YES — CRITICAL' if breakeven_risk else 'No'}.
```

def match_scenario(user_text: str) -> dict:
    """Find best matching cached scenario by keyword overlap."""
    text_lower = user_text.lower()
    best_match = None

        score = sum(1 for kw in scenario["trigger_keywords"] if kw in text_lower)

            best_match = scenario
    return best_match if best_match else CACHED_SCENARIOS["general_fault"]
